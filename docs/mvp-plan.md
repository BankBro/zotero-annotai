# Zotero AI Reader MVP 计划

## 1. MVP 定义

MVP 需要验证核心阅读闭环：

```text
选中 PDF 文本
-> 显示 翻译 / 解释 / 问答 操作
-> 打开可拖动、可调整大小的悬浮小窗
-> 调用 OpenAI-compatible Provider
-> 显示 AI 输出
-> 高亮选中文本
-> 写入结构化 AI 批注
```

如果这个闭环能稳定工作，插件的核心价值就已经被验证。

## 2. MVP 范围

包含：

- Zotero 7 插件骨架。
- 基础设置存储。
- OpenAI-compatible Provider Profile。
- Provider 连接测试。
- 翻译、解释、问答、批注总结的任务级模型选择。
- PDF 阅读器选中文本检测。
- 选区工具条，包含三个动作：
  - 翻译
  - 解释
  - 问答
- 通用悬浮小窗基础设施。
- 翻译小浮窗。
- 解释小浮窗。
- 临时问答小浮窗。
- 基础 AI 侧边聊天。
- 高亮选中文本并写入 annotation comment。
- 翻译、解释、问答的高亮颜色配置。
- AI 批注前缀。
- Zotero API 支持时提供基础 tag。
- 隐私模式设置。
- token 和上下文截断。
- 请求停止、重试、复制和错误清理。

不包含：

- 全文 RAG。
- 跨论文或文献库级别检索。
- 扫描版 PDF OCR。
- 图表图像理解。
- 将侧边聊天拖出成独立浮动面板。
- 高级长期记忆管理。
- 高级费用统计面板。
- 可视化 prompt 编辑器。
- 快捷键系统。

## 3. 工程阶段

### 阶段 0：文档与决策

交付物：

- 产品规格。
- MVP 计划。
- 架构文档。

退出标准：

- MVP 范围与后续阶段功能明确分离。
- 待验证技术问题已记录。

### 阶段 1：插件骨架

交付物：

- Zotero 7 插件 manifest。
- bootstrap 入口。
- 带命名空间的插件对象。
- 基础日志。
- 基础 preferences。
- 必要时增加构建和打包脚本。

退出标准：

- 插件可以安装到 Zotero 7。
- 能看到 startup 和 shutdown 日志。
- 可以读取和写入 preferences。

### 阶段 2：阅读器选区验证

交付物：

- 检测当前活跃 Zotero PDF 阅读器。
- 读取选中文本。
- 可获得时读取基础论文元数据。
- 获取选区位置或近似工具条锚点。

退出标准：

- 选中 PDF 文本后，插件代码可以拿到 selected text。
- 插件可以在选区附近显示临时调试 UI。
- 能处理选区丢失和空选区。

### 阶段 3：悬浮 UI 基础

交付物：

- 选区悬浮工具条。
- 通用 `FloatingPanel` 组件。
- 拖动支持。
- 调整大小支持。
- 关闭行为。
- loading、错误、重试、停止、复制和操作按钮状态。

退出标准：

- 翻译、解释、问答三个动作都可以打开悬浮小窗。
- 小浮窗不阻塞 PDF 阅读器正常操作。
- 翻译、解释和问答重复点击都会复用同一浮窗并更新当前选区上下文。
- 重复点击不移动已有浮窗，点击任意浮窗可见区域可以置顶。

### 阶段 4：Provider 与请求层

交付物：

- OpenAI-compatible client。
- 单个默认 Provider 设置 UI。
- API key 存储。
- 模型设置。
- 连接测试。
- 流式输出能力标记。
- 请求超时。
- 请求取消。
- 错误分类。

退出标准：

- 配置好的 Provider 可以通过连接测试。
- 连接测试只发送固定 ping prompt，不发送 PDF 选区文本。
- API key 不出现在日志或诊断状态里。
- 认证、模型、网络、超时、限流错误有可理解提示。
- 阶段三的选区工具条和浮窗行为不回退。

### 阶段 5：翻译流程

交付物：

- 收集选中文本和附近上下文。
- 渲染翻译输出。
- 通过 prompt 判断单词、短语或句子。
- 格式化批注输出。
- 创建高亮 annotation。
- 写入 annotation comment。

退出标准：

- 用户可以选中文本，点击“翻译”，查看结果并点击“加入批注”。
- 选中文本按翻译高亮颜色高亮。
- annotation comment 使用 `[AI-词义]` 或 `[AI-翻译]`。

### 阶段 6：解释流程

交付物：

- 收集更丰富的上下文。
- 渲染解释输出。
- 生成简洁批注。
- 生成详细批注。
- 创建高亮 annotation。
- 写入 annotation comment。

退出标准：

- 用户可以选中文本，点击“解释”，查看结果，并保存简洁或详细批注。
- annotation comment 使用 `[AI-解释-简洁]` 或 `[AI-解释-详细]`。

### 阶段 7：临时问答流程

交付物：

- 临时问答小浮窗。
- 同一 reader 内保留一个临时问答 thread。
- 问答浮窗保留独立多轮历史。
- 追问输入框。
- 将当前对话总结成简洁或详细批注。

退出标准：

- 用户可以选中文本，点击“问答”，进行多轮追问，并写入简洁或详细问答批注。
- 重复点击“问答”不会清空已有问答历史。

### 阶段 8：基础侧边聊天

交付物：

- 侧边聊天面板。
- 侧边栏宽度可调整。
- 按论文保存聊天历史。
- 识别当前选区。
- 基础的回答转批注动作。

退出标准：

- 用户可以打开侧边聊天，围绕当前论文提问，并从回答生成批注。
- 侧边聊天历史按论文持久化。

### 阶段 9：设置、隐私与打磨

交付物：

- 高亮颜色设置。
- 隐私模式设置。
- 上下文长度设置。
- token 限制设置。
- 批注覆盖策略。
- 首次使用云端 Provider 的提示。
- 诊断视图。

退出标准：

- 用户可以通过设置页配置 Provider、模型、颜色、隐私模式和限制。
- 用户手写批注不会被无确认覆盖。

## 4. 技术风险清单

### 高风险

- 从当前 PDF 选区创建高亮 annotation。
- 将 PDF 选区坐标映射到工具条位置。
- 向 Zotero PDF 阅读器注入 UI，同时避免和原生阅读器 UI 冲突。

### 中风险

- 创建后更新 annotation comment。
- 直接给 PDF annotation 打 tag。
- 稳定提取选中文本附近上下文。
- 在同一 reader 内维护单个问答浮窗及其会话状态。

### 低风险

- OpenAI-compatible API 调用。
- prompt 模板渲染。
- 基础设置持久化。
- token 截断。

## 5. MVP 默认决策

目标平台：

```text
Zotero 7
```

首个 Provider 类型：

```text
OpenAI-compatible
```

默认隐私模式：

```text
上下文模式：选中文本 + 附近上下文 + 可获得时的标题/摘要
```

默认批注覆盖策略：

```text
覆盖 [AI-...] 批注。
覆盖用户手写批注前确认。
```

默认上下文层级：

```text
翻译：L1
解释：L2/L3
问答小浮窗：L2 + 当前小浮窗历史
侧边聊天：L3
```

默认悬浮小窗行为：

```text
可拖动
可调整大小
手动关闭
不阻塞阅读器
```

## 6. 验收清单

当以下能力都能在 Zotero 7 中工作时，MVP 完成：

- 插件可以安装并启动。
- Provider 可以配置并测试。
- 可以检测 PDF 选中文本。
- 选区工具条显示“翻译 / 解释 / 问答”。
- 翻译小浮窗可以打开、调用 AI 并写入批注。
- 解释小浮窗可以打开、调用 AI 并写入简洁或详细批注。
- 问答小浮窗支持多轮问题，并写入简洁或详细问答批注。
- 基础侧边聊天可以回答问题并生成批注。
- 高亮颜色可以配置。
- AI 批注使用约定前缀。
- 用户手写批注不会被无确认覆盖。
- 请求可以停止、重试、复制。
- 基础隐私和 token 设置可用。
