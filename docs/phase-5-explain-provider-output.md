# 阶段五第三步：解释浮窗接入 Provider 输出

阶段五第三步用于验证 Zotero AnnotAI 可以把 PDF 选区文本发送给已配置的 OpenAI-compatible Provider，并在 `解释` 浮窗中显示真实解释结果。这个阶段仍不自动创建 annotation、不自动写入 comment、不自动高亮；`问答` 继续保持诊断壳。

## 测试包

```text
dist/zotero-annotai-0.1.20.xpi
```

## 解释请求行为

- 默认目标语言固定为中文。
- 请求只发送选中文本和论文标题、作者、日期等元数据。
- 不发送前后文，不发送整篇 PDF 文本，不做摘要或 RAG 检索。
- 解释结果要求包含 `核心意思：` 和 `补充说明：` 两个字段。
- 不记录 API key、Authorization header、完整 prompt、完整解释结果或完整选区文本。
- 每个请求使用当前 Provider 的 `timeoutMs`。

## 重复点击规则

- 同一 reader 内只保留一个解释浮窗。
- 重复点击不改变已有浮窗的位置和大小。
- 旧解释请求未返回时，如果用户对同一选区重复点击 `解释`，插件不发送第二个请求。
- 旧解释请求未返回时，如果用户选中新文本并点击 `解释`，插件会尝试 abort 旧请求，并立即发送新文本请求。
- 如果旧请求无法真正取消，旧结果返回后也会按 requestID 丢弃，不覆盖新文本结果。

## 手动验收

1. 安装 `dist/zotero-annotai-0.1.20.xpi` 并重启 Zotero。
2. Debug Output 中确认出现 `Startup 0.1.20`。
3. Provider 设置页和连接测试仍正常。
4. 打开 PDF，选中文本，点击 `解释`。
5. 解释浮窗应显示 `正在解释...`，随后显示真实解释结果、模型和耗时。
6. 解释结果包含 `核心意思：` 和 `补充说明：`，字段名在浮窗中加粗。
7. 解释完成前后都不自动创建 annotation、不高亮、不写 comment，也不显示 `写入批注`。
8. 成功状态下 `复制` 和 `重新解释` 正常；失败或超时时 `重试` 正常。
9. 请求未返回时重复点击同一选区的 `解释`，不应发送第二个请求。
10. 请求未返回时切换新文本并点击 `解释`，浮窗应立即切换到新文本，旧结果不应覆盖新结果。
11. `翻译` 的 Provider 输出和手动写入批注仍正常。
12. `问答` 仍显示诊断内容，不调用 AI。
13. Zotero 原生按钮、Translate for Zotero 和 AnnotAI 三个按钮都正常。
14. 三个 AnnotAI 浮窗可打开、拖动、缩放、关闭、置顶。
15. PDF 鼠标滚轮正常。
